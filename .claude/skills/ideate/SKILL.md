# Ideate — 웹 서비스 아이디어 리서치 스킬

## Trigger
- "아이디어 줘", "뭐 만들까", "어떤 서비스가 좋을까"
- "트렌드 조사", "인기 서비스 분석", "시장 분석"

## Steps

### Step 1: 방향 설정
- 사용자에게 관심 카테고리 확인 (없으면 전체 트렌드 조사)
- 타겟 시장 확인 (한국/글로벌)
- 개발 규모 확인 (1인/소규모 팀)

### Step 2: 트렌드 리서치
- WebSearch로 최신 웹 서비스/SaaS 트렌드 조사
- 최근 주목받는 서비스 유형, 기술 트렌드 파악
- 사용자 불만/페인포인트가 많은 영역 탐색

### Step 3: 경쟁 서비스 분석
- 유사 카테고리 상위 서비스 5~10개 분석
- 각 서비스의 핵심 기능, 강점, 약점 정리
- 차별화 기회 식별

### Step 4: 아이디어 도출
- 3~5개의 구체적 아이디어 제안
- 각 아이디어: 이름, 한줄 설명, 타겟, 핵심 기능, 차별점, 수익 모델, 기술 실현 가능성

### Step 5: 사용자 선택
- 아이디어 목록 제시 → 선택 요청
- `_workspace/01_idea_research.md`에 저장
